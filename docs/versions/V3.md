# V3 目标：售前与全链路平台化

更新时间：2026-03-21

## V3 的定位

V3 不是简单的功能堆叠，而是在 V1 的执行闭环和 V2 的项目交付管理之上，继续把 SCF 的完整服务链路平台化。

## V3 默认方向

### 1. 售前学生档案 / 线索管理

- 引入早于 `Enrollment` 的学生档案与沟通状态。
- 典型状态包括：
  - 沟通中
  - 已沟通
  - 需求梳理中
  - 方案制定中
  - 待报名
  - 已转报名

### 2. 服务全链路贯通

- 把售前、已报名服务、项目交付、排课、反馈、结课输出串成一条连续主线。
- 让一个学生的长期服务历史可以被完整回溯。

### 3. 更完整的输出物与归档

- 结课文书
- 进程文档
- 代码归档
- 拍摄资料
- 比赛材料

### 4. 多角色可视化

- Admin 视角：运营调度、交付进度、风险面板
- 老板视角：全局风险、节点到期、阶段分布
- 执行人视角：我负责的项目、我认领的节点、我即将到期的任务

### 5. 外部渠道与未来产品化

- 为未来的小程序、飞书、通知渠道保留接口边界
- 但仍坚持站内数据库与业务层为主事实源

## V3 先不锁死的内容

以下内容在 V3 只保留方向，不提前锁死实现：

- 具体 UI 形态
- 第三方集成细节
- 多租户或对外 SaaS 化细节
- 复杂 BI / 报表系统实现方式

## V3 的意义

V3 的目标不是让系统看起来更“大”，而是让 SCF 的真实服务模式变成一套可持续复用、可交接、可扩展的平台能力。
