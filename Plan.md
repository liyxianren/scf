# 创意项目生成 Agent (Creative Project Generator) - 开发计划

## 1. 项目背景与目标
**背景**: SCF 致力于为国际学校高中生提供定制化辅导。在商业竞赛指导中，学生常提出特定但模糊的需求（如：人设+关键词）。
**痛点**: 公司内部并非所有员工都擅长使用 AI 工具进行头脑风暴，导致创意产出效率不一。
**目标**: 打造一个内部使用的 "创意生成 Agent"，只需输入简单的标签和要求，即可自动产出高可行性、有创意的项目选题方案。

## 2. 核心功能设计

### 2.1 用户输入 (Input)
员工在前端界面填写以下信息：
1.  **目标赛事** (可选): 例如 Diamond Challenge, CTB, USAD, 丘成桐奖等。
2.  **学生画像/人设**: 例如 "擅长编程的环保主义者"、"热爱心理学的社恐患者"。
3.  **核心关键词**: 例如 `环保` + `APP` + `AI` + `新想法`。
4.  **额外要求**: 例如 "需要软硬件结合" 或 "侧重商业模式创新"。

### 2.2 Agent 输出 (Output)
Agent 应生成 3-5 个不同维度的项目提案，每个提案包含：
1.  **项目名称**: 具有吸引力的商业命名。
2.  **Slogan/一句话介绍**: 核心价值主张。
3.  **痛点分析**: 该项目解决了什么具体问题（Why Now?）。
4.  **解决方案/产品形态**: APP/网站/硬件的具体功能描述。
5.  **创新点/差异化**: 相比市面上现有产品的优势。
6.  **技术/执行可行性**: 针对高中生的实施难度评估。

## 3. 技术架构方案

### 3.1 大模型底座
- **模型**: ChatGLM (智谱 AI)
- **集成**: 通过 API 接入后端。
- **知识库**: (可选) 后续可挂载 SCF 的历史成功案例库，利用 RAG 增强生成效果。

### 3.2 Agent 节点设计 (Node Workflow)

我们要建立一个包含 4 个核心节点的链式工作流 (Chain of Thought)：

#### 1. 需求拆解与扩充节点 (Input Analysis Node)
- **功能**: 将用户输入的简短关键词（如"环保+APP"）扩充为具体的**赛道方向**。
- **目的**: 避免生成过于泛泛的点子，确保后续发散的方向多样化。
- **输出示例**: 
  - 方向A: 社区垃圾分类激励平台
  - 方向B: 濒危物种识别图鉴
  - 方向C: 碳足迹个人追踪器

#### 2. 头脑风暴节点 (Brainstorming Node)
- **功能**: 基于上述方向，进行发散思维，生成 6-9 个原始创意雏形。
- **提示词策略**: 鼓励"天马行空"，暂时忽略限制，追求创意的新颖性。

#### 3. 可行性评估节点 (Feasibility Assessor Node) - **核心风控**
- **功能**: 扮演"技术总监"角色，对原始创意进行严格的**边界检查**。
- **评估准则 (Boundary Rules)**:
  - **软件类 (APP/Web)**: ✅ **高可行性**。判定为"可执行"，因为公司可借助 AI 辅助完成这类开发。
  - **硬件类**: ⚠️ **中可行性**。需严格审查合理性，排除需要昂贵工业设备或非标零件的想法。
  - **实验类 (生物/化学)**: ❌ **低可行性**。直接判定为"不可行"或大幅扣分，明确标注"公司无法支持湿实验环境"。
- **动作**: 剔除不可行方案，保留 Top 3。

#### 4. 方案细化节点 (Detailing Node)
- **功能**: 将通过筛选的 Top 3 点子包装成正式的商业计划书雏形。
- **输出**: 包含 Slogan、痛点分析、产品功能列表、技术栈建议的完整报告。

### 3.3 Prompt 策略与详细设计 (Detailed Prompts)

以下是各节点的详细 Prompt 模板，开发时需替换 `{{variable}}` 为实际内容。

#### Node 1: 需求拆解与扩充 (Input Analysis)
```markdown
# Role
资深国际课程规划师，擅长将模糊的学生兴趣转化为具体的竞赛赛道。

# Task
用户提供了一些关键词：{{user_keywords}}
以及学生画像：{{student_profile}}

# Constraints
1. 你的任务是扩展思路，不要局限于字面意思。
2. 给出 3 个截然不同的**赛道方向**（Direction）。
3. 赛道必须适合高中生参加的商业竞赛（如 Diamond Challenge）。

# Output Format (JSON)
{
  "directions": [
    "方向1：...",
    "方向2：...",
    "方向3：..."
  ]
}
```

#### Node 2: 头脑风暴 (Brainstorming)
```markdown
# Role
硅谷创业公司的创意总监，思维活跃，擅长提出颠覆性的点子。

# Task
基于以下 3 个赛道方向，分别生成 3 个具体的项目创意（共 9 个）：
{{directions}}

# Guidelines
- 鼓励“微创新”，将现有技术应用在非传统领域。
- 此时**不需要**考虑过多的实现难度，优先保证创意的新颖性。
- 每个创意用一句话描述（<15字）。

# Output Format (List)
- 方向1创意A: ...
- 方向1创意B: ...
...
```

#### Node 3: 可行性评估 (Feasibility Assessor)
```markdown
# Role
SCF 公司的技术总监，负责评估高中生项目的落地可行性。

# Context
我们公司可以提供软件开发支持（APP/Web），但无法提供生物/化学湿实验环境。硬件开发仅限于开源硬件（Arduino/树莓派）。

# Task
对以下创意列表进行打分和筛选：
{{raw_ideas}}

# Scoring Rules (CRITICAL)
1. **软件类 (纯APP/网站/数据分析)**: 
   - 可行性得分: 9-10分。
   - 评语: "开发可控，AI可辅助"。
2. **轻量级硬件类 (基于现有传感器/模块)**: 
   - 可行性得分: 6-8分。
   - 评语: "需评估硬件成本和调试难度"。
3. **重型硬件/工业制造**: 
   - 可行性得分: 0-4分。
   - 评语: "超出高中生能力，需工厂配合，不可行"。
4. **生物/化学/医学实验 (需实验室)**: 
   - 可行性得分: 0分。
   - 评语: "REJECT: 公司无实验室环境"。

# Output
请输出综合得分最高的 **Top 3** 创意的名称，并附带简短的通过理由。
```

#### Node 4: 方案细化 (Detailing)
```markdown
# Role
商业计划书撰写专家。

# Task
将以下 3 个入选创意包装成专业的项目提案：
{{selected_ideas}}

# Requirement
针对每个创意，生成以下内容（Markdown格式）：
1. **项目名称**: 商业化、朗朗上口的名字。
2. **Slogan**: 一句打动评委的口号。
3. **痛点 (Why Now)**: 为什么现在需要这个东西？
4. **解决方案 (Product)**: 具体是个APP还是什么？核心功能有哪3点？
5. **技术栈 (Tech)**: 比如 "Python + Flutter + ChatGLM API"。
6. **商业价值**: 怎么赚钱或产生社会影响力？

# Tone
专业、自信、逻辑清晰。
```
