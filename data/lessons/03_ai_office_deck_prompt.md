## Prompt 指挥 Agent 做办公产物

前两节我们讲了"Agent 是什么"和"通用 Agent 框架怎么设计"。这一节回到实操：

- **怎么写一个"能用"的 Prompt？** 为什么同样的需求，别人让 AI 做的 PPT 能用，你的不能用？
- **PPT / Word / Excel 各有什么模板？** 能不能直接 copy 回去用？
- **怎么让 Agent 的产出从 60 分跳到 90 分？** 三个进阶技巧
- **哪些坑最常踩？** 5 个误区 + 一句话收尾

这一节讲完，你应该能拿着 Codex、Claude Code 或 Gemini，独立做出可以用的 PPT、Word、Excel。

> 好消息：这件事不需要你会编程。你需要会的，是**把需求讲清楚**。

---

## 办公场景的本质：结构化产物生成

我们每天做的 PPT、Word、Excel，仔细看都在做**同一件事**——把非结构化的想法，变成结构化的文件：

| 产物 | 本质结构 |
|---|---|
| **PPT** | 页的列表，每页有标题 + 要点 + 可选的图/表 |
| **Word** | 章节树（H1 / H2 / H3），每节有段落、表格、引用 |
| **Excel** | 行列矩阵，加上公式、图表 |

一旦你意识到"办公文档 = 有规律的结构"，你就明白了一件关键的事：

> AI 做这些事的关键，不是它的"文采"，是你能不能讲清楚**你要的结构长什么样**。

结构讲清楚了，Agent 填内容就稳；结构讲不清，产出就会是"看起来对但用不了"。

---

## Prompt 的 5 个构件（总览）

<img src="/static/images/ai-office/prompt-anatomy.svg" alt="Prompt 5 构件" />

任何办公场景的 Prompt，都可以拆成 5 个构件。**缺哪一块，就在哪个维度翻车**：

- **① 角色（Role）**——你是一个 ___
- **② 任务（Task）**——要做什么：动词 + 对象 + 数量
- **③ 约束（Constraint）**——字数 / 风格 / 格式 / 禁止项
- **④ 素材（Material）**——上下文 / 数据 / 参考文件
- **⑤ 产出格式（Output）**——返回什么类型的文件 + 结构样板

后面 5 页我们逐个讲。记住这 5 个词，下次写 Prompt 就有骨架。

> 你会发现——90% "AI 产出烂"的问题，都是这 5 个构件里某一个没写清楚。

---

## 构件①：角色 —— 给模型身份锚点

Prompt 的第一句话永远应该是"你是一个 ___"。**身份锚点会调用模型不同的表达风格、专业术语和常识储备**。

对比一下：

- ❌ "帮我写份销售分析" ——模型会给你一段通用的、没有专业感的文字
- ✅ "你是一个在快消行业做过 10 年销售分析的咨询顾问。帮我写份销售分析" ——同一个模型的产出会明显更专业、更有结构、更少通用套话

为什么这样有效？因为模型见过海量文字，它知道"咨询顾问"写的东西长什么样。你给它一个身份，它就调用那类文字的模式。

常用角色池：

- PPT 场景：资深 PPT 设计师 / 做过 10 年咨询的 McKinsey 顾问
- Word 场景：技术文档工程师 / 法务合同审查员 / 产品经理
- Excel 场景：数据分析师 / 财务分析师

> 写 Prompt 的第一行，永远是"你是一个 ___"。这是最低成本、收益最高的一招。

---

## 构件②：任务 —— 动词 + 对象 + 数量

很多人写 Prompt 会写成"帮我搞一下 XX"这种废话。**一个清晰的任务描述至少要有三个要素：动词、对象、规模**。

对比：

- ❌ "帮我搞一下 PPT"
- ✅ "生成一份 8 页的销售复盘 PPT"
- ✅✅ "生成一份 8 页的销售复盘 PPT，包括：1 页封面、1 页议程、5 页核心数据、1 页总结"

差距在哪？后者把**结构**也讲清楚了——AI 不用猜你要什么结构。

一个小诀窍：**任务里包含数字，产出质量会飙升**。

- 页数、字数、条目数
- 时间范围、版本号
- 多少行 × 多少列

数字让任务边界清晰，AI 不再"发挥"。

> 不让 AI 猜，它就不乱编——这是控制产出质量的第一步。

---

## 构件③：约束 —— 收益最大的一块

**这是 Prompt 里最容易被忽视、但收益最大的一块**。约束越具体，模型发挥越稳定。

常见的约束维度：

- **字数**：每页标题 ≤15 字，正文每条 bullet ≤20 字
- **风格**：语气克制专业，避免口语化
- **格式**：用 Markdown 表格呈现，不要嵌套列表
- **禁止项**：禁止虚构数据；禁止使用 "我相信 / 我认为" 等主观表达
- **受众**：受众是技术总监，不需要解释基础概念
- **语言**：全文中文，英文专有名词保留原文

任何办公 Prompt，**至少加这 3 条约束**：

1. 字数 / 条目数上限
2. 禁止虚构（数据类任务尤其关键）
3. 语气风格（避免太"AI"味）

> 没有约束的 Prompt，就像没有边界的画布——AI 会把每个细节都往"平均美感"上拉，结果平庸。

---

## 构件④：素材 —— 不给素材 Agent 只能编

**不给素材，Agent 只能靠训练记忆和编造**。给素材的方式有几种：

- **直接粘贴**：短的数据直接贴到 Prompt 里
- **附带文件**：让 Agent 读本地文件，比如 `"请读取 ./sales_q1.csv"`
- **参考样板**：给一份好的过往产出，比如 `"参考这份去年的复盘 PPT 的结构"`
- **引用链接**：对于能联网的 Agent，贴 URL 让它去读

一个关键的约定——**素材和指令要明确分开**。

不要写"按去年的标准来"——AI 不知道"去年的标准"是啥，只能瞎猜。要写"参考这份文件 `./2024_q4_report.pdf`，按它的章节结构和语气写"——AI 会真的去读。

> 对 AI 来说：你描述过的事 ≠ 你给过它的材料。**引用必须变成文件或粘贴，否则它会编**。

---

## 构件⑤：产出格式 —— 最容易被忘记

**这是最容易被忘记的一块，也是"AI 产出不能用"的头号原因**。

你说"帮我做个 PPT"，Agent 可能给你一段 Markdown——因为 Markdown 是 LLM 最熟悉的产出格式。这不是 AI 蠢，是**你没说要 `.pptx` 文件**。

正确写法：

- ✅ "产出：调用 pptx Skill 生成 output.pptx 文件"
- ✅ "产出：直接生成 .xlsx 文件保存到 ./out/report.xlsx"
- ✅ "产出：返回一段 JSON，字段是 `{title, bullets[], chart_data}`，后续我自己渲染"

产出格式里**顺便塞一个样板**，效果会更好：

```
产出格式：JSON，结构如下：
{
  "slides": [
    {"title": "...", "bullets": ["...", "..."], "chart": {...}}
  ]
}
```

> 养成一个习惯：**Prompt 的最后一行永远写产出格式**。

## 办公闭环：从需求到产物

<img src="/static/images/ai-office/office-workflow.svg" alt="办公实战闭环" />

把 5 构件套到实际流程里，整个闭环是这样的：

- **① 用户需求**——你讲的人话（"我要一份 Q1 销售复盘 PPT"）
- **② Prompt 构造**——按 5 构件展开成结构化 prompt
- **③ Agent 规划**——Agent 识别要挂哪个 Skill、读哪些文件、出什么大纲
- **④ 工具执行**——Agent 在沙箱里调用 python-pptx / openpyxl / matplotlib
- **⑤ 产物**——实际的 .pptx / .docx / .xlsx 文件

**你作为使用者只出现在 ①②⑤**，③④ 是 Agent 的活。所以 Prompt 质量 ≈ 产物质量——你就只有说那一次话的机会。

如果产出不满意，回到 ② 调整 Prompt，再跑一轮。

> Agent 不是魔术师，它是个严格按你指令做事的员工。员工产出好不好，看指令写得清不清。

## 三大模板之一：PPT

下面这个模板，把上面讲的 5 构件全部落到了实处。改三个字段（角色、主题、素材）就能直接用：

```
你是一个在咨询行业做过 10 年 PPT 的资深顾问。

任务：基于以下要点，生成一份 8 页的 {主题} PPT。

页面结构：
- 第 1 页：标题封面（主题 + 副标题 + 日期）
- 第 2 页：议程（3-5 条，对应后面页面）
- 第 3-7 页：每页一个核心主题，包含：
    · 页标题（一句话论点）
    · 3-5 个 bullet 支撑这个论点
    · 如有数据，每页附一张图/表
- 第 8 页：总结 + 3 条行动项

约束：
- 每页标题 ≤ 15 字，要是完整的论点，不要是名词短语
- 每个 bullet ≤ 20 字
- 语气克制专业，不要用 emoji 不要用感叹号
- 禁止虚构数据；数字都要来自素材
- 配色：主色 #2563eb，背景白色

素材：
{粘贴你的要点 / 数据 / 上下文}

产出：调用 pptx Skill 生成 output.pptx 保存到 ./out/
```

> 把这个模板存到你的 Prompt 库里，下次做任何 PPT 都从这里开始改。

## 三大模板之二：Word

Word 的结构是章节树，关键约束和 PPT 不同：

```
你是一个技术文档工程师。

任务：基于以下素材，生成一份 {主题} 文档。

文档结构：
# {主题}
## 1. 背景
## 2. 现状分析（含 1 张对比表格）
## 3. 方案描述
  ### 3.1 方案 A
  ### 3.2 方案 B
## 4. 风险与缓解
## 5. 结论与下一步

约束：
- 总字数 2000-3000 字
- 每个章节的首句是论点，其余是支撑
- 章节 2 必须有一张 3 列以上的对比表格
- 禁止 "首先 / 其次 / 再者" 这种流水账式连接词
- 禁止 "我们应该 / 我相信" 等主观表达
- 专业术语首次出现时括注英文

素材：{粘贴素材}

产出：调用 docx Skill 生成 output.docx，包含目录和页眉
```

Word 场景最容易翻车的是**表格质量**。一定要求表格有明确的对比维度，别让它自由发挥。如果公司有模板（字体、段落间距），把模板 docx 作为素材扔进去让它参考。

> 法务 / 合规类文档，一定要写"禁止虚构法律条文 / 案例"，否则会出大事。

## 三大模板之三：Excel

Excel 的结构是 Sheet + 行列 + 公式：

```
你是一个数据分析师。

任务：基于以下数据，生成一份 {主题} Excel 报表。

Sheet 结构：
- Sheet1: 原始数据（保留所有字段）
- Sheet2: 汇总（按 {维度} 分组，含 SUM / AVG / COUNT 三列）
- Sheet3: 透视表（行：{行维度}，列：{列维度}，值：{度量}）
- Sheet4: 图表（柱状图 + 折线图各一，数据源是 Sheet2）

约束：
- 所有金额保留 2 位小数，加千分位分隔符
- 表头加粗 + 浅灰背景（#f1f5f9）
- 所有公式必须是 Excel 公式，不要是硬编码的值
- 日期列统一格式 YYYY-MM-DD

数据：{粘贴 CSV 或指定文件路径}

产出：调用 xlsx Skill 生成 output.xlsx
```

Excel 最容易翻车的是**公式变硬编码**——AI 可能给你一个 `"销售额：125000"` 而不是 `"=SUM(B2:B10)"`。一定要在约束里强调"必须用公式"。

> 做出来的 Excel，一定要打开看公式栏——文字对了不等于公式对了。

## 三个让产出翻倍的技巧

产出从 60 分到 90 分，靠的不是更长的 Prompt，而是这三招：

**技巧一：分阶段指挥**

做一份 15 页的完整提案 PPT：
- ❌ 错误做法：一次 Prompt 让 AI 直接出完整 15 页——前几页还行，后面越水；你想改一点就全重跑
- ✅ 正确做法：先出**大纲**（每页一行标题）→ 你 review → 再**填每页 bullet** → 再 review → 最后**生成文件**

每阶段都有人类对齐点，错误不会累积。

**技巧二：示例驱动（few-shot）**

AI 听不懂"专业"、"克制"、"得体"这种抽象词，但**能看懂具体例子**。写 Prompt 时贴一个你以前写过的好内容，说"按这个风格写新的"——比你描述 10 句"请用专业风格"管用十倍。

**技巧三：让 AI 自审**

跑完一轮后，把你的约束清单再扔回去："回顾你刚才生成的 PPT，逐条检查是否符合以下约束：每页标题 ≤15 字？禁止虚构数据？配色符合？" AI 非常擅长自检，往往它自己能挑出并改掉 3-5 个错误。

> 分阶段 + 示例驱动 + 自审，这三个技巧独立做每个都能提升 20 分。

## 5 个常见误区 + 一句话收尾

最后一页，5 个最常见的翻车点：

- **误区一：Prompt 越长越好**——错。长 Prompt 里的噪音会稀释关键约束。目标是**信息密度高**，不是字数多
- **误区二：让 AI 一次做完复杂任务**——错。复杂任务一定要分阶段，每阶段对齐
- **误区三：忘写产出格式**——错。不写 `.pptx` 它就给你 Markdown。Prompt 最后一行永远写产出格式
- **误区四：素材靠引用不靠粘贴**——错。"按我们公司标准" ≠ 给素材。粘贴或附件才算数
- **误区五：拿到产物不检查**——错。AI 会自信地犯错，数字可能不对、结构可能走样，一定要人类 review

**一句话收尾**——

> 办公自动化的不是 AI，是**你 + AI**。你给结构、给约束、给素材，AI 执行。你讲得清楚，它就做得漂亮；你讲得含糊，它就帮你平庸。
