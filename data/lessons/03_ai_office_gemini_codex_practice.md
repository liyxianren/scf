## 1. 办公场景的本质：结构化产物生成

我们每天做的 PPT / Word / Excel，仔细看都在做**同一件事** —— **把非结构化的想法，变成结构化的文件**。

| 产物 | 本质结构 |
|---|---|
| **PPT** | 页（Slide）列表，每页有标题 + 要点 + 可选的图/表 |
| **Word** | 章节树（H1 / H2 / H3），每节有段落、表格、引用 |
| **Excel** | 行列矩阵，加上公式、图表 |

一旦你意识到"办公文档 = 有规律的结构"，就能明白：

> **AI 做这些事情的关键，不是它的文采，而是你能不能讲清楚"你要的结构长什么样"。**

结构讲清楚了，Agent 填内容就稳；结构模糊不清，它产出的就是"看起来对但用不了"的东西。

## 2. 一个"能用"的 Prompt 的 5 个构件

<img src="/static/images/ai-office/prompt-anatomy.svg" alt="Prompt 的 5 个构件" style="width:100%;max-width:900px;"/>

任何办公场景的 Prompt，缺哪一块就会在对应维度翻车。先把这 5 个词记死：

### 2.1 角色（Role）

> "你是一个 ___"

给模型一个**身份锚点**。不同的身份会调用它不同的表达风格、专业术语、常识储备。

**对比**：

- ❌ `"帮我写份销售分析"`
- ✅ `"你是一个在快消行业做过 10 年销售分析的咨询顾问。帮我写份销售分析。"`

后者的输出会明显更专业、更有结构、更少"通用套话"。

**常用角色**：

- PPT：`资深 PPT 设计师` / `做过 10 年咨询的 McKinsey 顾问`
- Word：`技术文档工程师` / `法务合同审查员` / `产品经理`
- Excel：`数据分析师` / `财务分析师`
- 邮件：`英文邮件撰写专家` / `B2B 销售`

### 2.2 任务（Task）

> "要做什么（动词 + 对象 + 数量）"

最容易被写成"帮我搞一下 XX"这种废话。**动词 + 对象 + 规模**三个要素都要有。

**对比**：

- ❌ `"帮我搞一下 PPT"`
- ✅ `"生成一份 8 页的销售复盘 PPT"`
- ✅✅ `"生成一份 8 页的销售复盘 PPT，包括：1 页封面、1 页议程、5 页核心数据、1 页总结"`

**诀窍**：任务里包含**页数 / 字数 / 条目数 / 时间范围**等数字，产出质量就会飙升。

### 2.3 约束（Constraint）

> "字数 / 风格 / 格式 / 禁止项"

**这是收益最大但最容易被忽视的一块。**

约束越具体，模型发挥越稳定。常见的约束维度：

| 维度 | 例子 |
|---|---|
| 字数 | `"每页标题 ≤15 字，正文每条 bullet ≤20 字"` |
| 风格 | `"语气克制专业，避免口语化，不要用 emoji"` |
| 格式 | `"用 Markdown 表格呈现，不要用嵌套列表"` |
| 禁止项 | `"禁止虚构数据；禁止使用'我相信 / 我认为'等主观表达"` |
| 受众 | `"受众是技术总监，不需要解释基础概念"` |
| 语言 | `"全文中文，英文专有名词保留原文"` |

**一个黄金模板**：任何办公 Prompt 都至少加这三条约束：

1. 字数 / 条目数上限
2. 禁止虚构（特别是数据类任务）
3. 语气风格（避免太 AI）

### 2.4 素材（Material）

> "上下文 / 数据 / 参考文件"

**不给素材，Agent 只能靠训练记忆和编造。**

给素材的方式：

- **直接粘贴**：短的数据直接贴到 prompt 里
- **附带文件**：让 Agent 读本地文件（`"请读取 sales_q1.csv"`）
- **参考样板**：给一份好的过往产出（`"参考这份去年的复盘 PPT 的结构"`）
- **引用链接**：对于能联网的 Agent，贴 URL 让它去读

**重要约定**：素材和约束要明确分开。不要把"参考这份去年的 PPT"写成"按去年的标准来"——前者 Agent 会真的去读，后者 Agent 只能瞎猜"去年的标准"是啥。

### 2.5 产出格式（Output Format）

> "返回什么类型的文件 + 结构样板"

**这是最容易被忘记的一块，也是最经常让人骂"AI 真蠢"的原因。**

你说"帮我做个 PPT"，Agent 默认可能会给你 Markdown，因为 Markdown 是 LLM 最熟悉的产出格式。这是**你的问题**，不是 Agent 的问题——你没说要 `.pptx` 文件。

**正确写法**：

- ✅ `"产出：调用 pptx Skill 生成 output.pptx 文件"`
- ✅ `"产出：返回一段 JSON，字段是 {title, bullets[], chart_data}，后续我会自己渲染"`
- ✅ `"产出：直接生成 .xlsx 文件保存到 ./out/report.xlsx"`

**产出格式里顺便塞一个样板**，效果会更好：

```
产出格式：JSON，结构如下：
{
  "slides": [
    {"title": "...", "bullets": ["...", "..."], "chart": {...}},
    ...
  ]
}
```

## 3. 办公闭环：从需求到产物

<img src="/static/images/ai-office/office-workflow.svg" alt="办公实战闭环" style="width:100%;max-width:900px;"/>

把第 2 节的理论套到实战，整个闭环长这样：

1. **① 用户需求**：你讲的人话（"我要一份 Q1 销售复盘 PPT"）
2. **② Prompt 构造**：按 5 构件展开成结构化 prompt
3. **③ Agent 规划**：Agent 自动识别该挂哪个 Skill、读哪些文件、出什么大纲
4. **④ 工具执行**：Agent 在沙箱里调用 `python-pptx` / `openpyxl` / `python-docx`
5. **⑤ 产物**：实际的 `.pptx` / `.docx` / `.xlsx` 文件
6. **⑥（可选）反馈**：不满意 → 指出具体问题 → 回到 ②

**你作为使用者**，只出现在 ①②⑤⑥ 四个点上。③④是 Agent 的活。这也是为什么 **Prompt 的质量决定了产出的质量** —— 你就只有这一次说话的机会。

## 4. 三大产物的 Prompt 模板

下面三个模板可以直接 copy、改字段、用。

### 4.1 PPT 模板（pptx）

```
你是一个在咨询行业做过 10 年 PPT 的资深顾问。

任务：基于以下要点，生成一份 8 页的{主题}PPT。

页面结构：
- 第 1 页：标题封面（主题 + 副标题 + 日期）
- 第 2 页：议程（3-5 条，对应后面页面）
- 第 3-7 页：每页一个核心主题，包含
    · 页标题（一句话论点）
    · 3-5 个 bullet 支撑这个论点
    · 如有数据，每页附一张图/表
- 第 8 页：总结 + 3 条行动项

约束：
- 每页标题 ≤ 15 字，要是完整的论点，不要是名词短语
- 每个 bullet ≤ 20 字
- 语气克制专业，不要用 emoji 不要用感叹号
- 禁止虚构数据；数字都要来自素材
- 配色：主色 #2563eb，背景白色，文字 #1e293b

素材：
{粘贴你的要点 / 粗数据 / 上下文}

产出：
调用 pptx Skill，生成 output.pptx 文件保存到 ./out/ 目录
```

**使用贴士**：

- 如果第一轮产出页数不对，直接说"页数不对，请严格按 8 页"通常一次就改好
- 图表细节不要在第一轮塞太多，让它先出骨架，第二轮再让它加图
- 拿到产物后**一定要打开 PPT 看一遍**，别只看 Agent 的文字摘要——有时候文件内部和它描述的不一致

### 4.2 Word 模板（docx）

```
你是一个技术文档工程师。

任务：基于以下素材，生成一份{主题}文档。

文档结构：
# {主题}

## 1. 背景
...

## 2. 现状分析（含 1 张对比表格）
...

## 3. 方案描述
### 3.1 方案 A
### 3.2 方案 B

## 4. 风险与缓解

## 5. 结论与下一步

约束：
- 总字数 2000-3000 字
- 每个章节的首句是论点，其余是支撑
- 章节 2 必须有一张 3 列以上的对比表格
- 禁止"首先 / 其次 / 再者"这种流水账式连接词
- 禁止"我们应该 / 我相信"等主观表达
- 专业术语首次出现时括注英文

素材：
{粘贴素材}

产出：
调用 docx Skill，生成 output.docx 文件，包含目录和页眉
```

**使用贴士**：

- Word 的**表格质量**是最能看出 Agent 功力的地方。要求表格必须有明确的对比维度（别让它自由发挥）
- 如果公司有模板（字体 / 段落间距 / 页边距），把模板 docx 作为素材扔进去，让 Agent 读完再生成
- 法务 / 合规类文档一定要标注"禁止虚构法律条文 / 案例"，否则会翻车

### 4.3 Excel 模板（xlsx）

```
你是一个数据分析师。

任务：基于以下数据，生成一份{主题} Excel 报表。

Sheet 结构：
- Sheet1: 原始数据（保留所有字段）
- Sheet2: 汇总（按{维度}分组，含 SUM / AVG / COUNT 三列）
- Sheet3: 透视表（行：{行维度}，列：{列维度}，值：{度量}）
- Sheet4: 图表（柱状图 + 折线图各一，数据源是 Sheet2）

约束：
- 所有金额保留 2 位小数，并加千分位分隔符
- 表头加粗 + 浅灰背景（#f1f5f9）
- 所有公式必须是 Excel 公式，不要是硬编码的值
- 如果数据里有缺失值，第一行注明处理方式（填 0 还是跳过）
- 日期列统一格式 YYYY-MM-DD

数据：
{粘贴 CSV 或者指定文件路径}

产出：
调用 xlsx Skill，生成 output.xlsx 文件
```

**使用贴士**：

- Excel 最容易翻车的是**公式写成硬编码**（比如 `"销售额: 125000"` 而不是 `"=SUM(B2:B10)"`）。一定要在约束里强调"必须用公式"
- 透视表要求最好明确写"行/列/值"三个维度，否则 Agent 会瞎选
- 图表数据源要指定是哪个 Sheet 的哪块区域，不然图可能指向空白

## 5. 进阶技巧（让产出质量翻一倍）

### 5.1 分阶段指挥（最实用的技巧）

**场景**：做一份 15 页的完整提案 PPT。

**错误做法**：一次 prompt 让 Agent 直接出完整 15 页。结果是：前几页还行，后面越来越水；你想改个局部，它重新生成又全打乱。

**正确做法 —— 三阶段**：

**阶段 1：先出大纲，人类 review**

```
现在只生成 PPT 大纲（每页一行标题），不要填内容。
```

你 review 大纲，增删修改，直到骨架你满意。

**阶段 2：逐页填充**

```
基于刚才的大纲，现在开始填充每一页的 bullet。
先从第 3-5 页开始（核心论证页），其他页先不动。
```

这次只做一部分，你 review 完满意再做下一批。

**阶段 3：整合 + 生成文件**

```
所有页的 bullet 都 OK 了，现在调用 pptx Skill 生成完整文件。
```

**为什么有效？** 因为每一阶段都有**人类对齐点**，错误不会累积到最后。

### 5.2 示例驱动（few-shot）

Agent 听不懂抽象的"专业"、"克制"、"得体"，但能**看懂具体的例子**。

**写法**：

```
参考这个风格（以下是一段我写过的好的开头）：

> "Q1 我们面对三个挑战：市场增速放缓、竞品价格战、核心客户流失。
>  本次复盘聚焦三个问题：哪些假设被证伪、哪些执行动作奏效、Q2 如何调整。"

按这个风格写 {新的任务} 的开头。
```

这比你说"请用咨询公司的写作风格"有效十倍。**让 Agent 模仿具体的例子，是成本最低、收益最高的技巧。**

### 5.3 Code Review 思维：让 Agent 自查

跑完一轮后，把你的**约束清单**再扔给它，让它**自己检查有没有违反**：

```
回顾你刚才生成的 PPT，检查是否符合以下约束：
1. 每页标题 ≤ 15 字？
2. 每个 bullet ≤ 20 字？
3. 是否有任何虚构数据？
4. 配色是否都是 #2563eb？

逐条回答"符合 / 不符合 + 哪里不符合"，然后修改不符合的地方。
```

Agent 非常擅长这类**自检**任务。往往它自己能挑出并改掉三五个之前犯的错——而你完全不用人眼去扫一遍。

### 5.4 变量化：让你的好 Prompt 可以复用

写出来的 Prompt 不要用一次就扔。**把里面会变的部分参数化**：

```
角色：{role}
主题：{topic}
素材：{material}
页数：{page_count}
```

存在自己的 Prompt 库里，下次做类似任务直接套模板，只改变量。**一年下来你会积累几十个可复用的高质量 Prompt**，这是个人办公效率跃迁的关键。

## 7. 常见误区与坑

### 误区一：Prompt 写得越长越好

**错。** 长 prompt 里的噪音会稀释关键约束。目标是**信息密度高**，不是**字数多**。一条 200 字但每条都有用的 prompt，比一条 800 字啰嗦重复的 prompt 效果好得多。

**原则**：每写完一条约束，问自己"如果删掉它，Agent 会不会在这个点上出错？" 删掉后不会出错的，就砍掉。

### 误区二：让 Agent 一次做完复杂任务

**错。** 见 5.1 节。复杂任务**一定要拆阶段**。AI 的错误是会累积的，每一步对齐一次，最终产出才稳。

### 误区三：忽略产出格式

**错。** 你说"帮我做 PPT"，Agent 给你 Markdown 是它默认的行为。它不是在糊弄你，是你没说要 `.pptx`。

**习惯**：Prompt 最后一行永远写"产出 XX 格式 / XX 类型的文件"。

### 误区四：素材用引用而不是粘贴

**错（这点很隐蔽）。** 你说"按我们公司标准写"，Agent 并不知道"我们公司标准"是什么。素材必须是**模型能直接读到的内容**——粘贴文本、附上文件、或者告诉 Agent 去读具体路径。

### 误区五：拿到产物不检查

**错。** AI 会很自信地告诉你"我已完成"，但产物里可能有各种问题（数字错、结构走样、虚构内容）。**永远要人类 review**，尤其是对外发送或落档的文件。

> **讲课提示汇总（学生版可删）**
>
> - 本节最重要的三个顿悟：
>   1. 办公文档 = **结构化产物**，Prompt 的核心工作是讲清结构
>   2. Prompt 5 构件（角色 / 任务 / 约束 / 素材 / 产出格式）缺哪块在哪翻车
>   3. 分阶段指挥 + 示例驱动 + 自审，是让产出上一个台阶的三连击
>
> - **时间分配建议**：总共 40-50 分钟（含现场 Demo）
>   - 5 构件讲解：10 分钟
>   - 三个模板快速过：10 分钟
>   - 进阶技巧：5 分钟
>   - 现场 Demo（双跑对比）：15-20 分钟
>   - Q&A：5-10 分钟
>
> - **现场 Demo 避坑**：
>   - 演示前自己**预跑一遍**，别现场才发现模型在闹脾气
>   - 演示素材**简单具体**（一份 10 行 CSV，一段 100 字需求描述），别搞成复杂数据现场卡壳
>   - 演示时**有意先犯错再改对**，学员看到"坏 Prompt → 坏结果 → 加约束 → 好结果"的对比会更深刻
>   - 准备 1-2 个 Backup Demo，万一网络或模型挂了能兜底
>
> - **结尾建议**：让大家**现场试一次**自己的办公场景（做一份本周邮件 / 一份周报 / 一份对账表），这比你再讲 10 分钟理论都有效
