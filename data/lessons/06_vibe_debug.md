# 当AI出错了怎么办

---

## AI会出错吗？

会！而且经常出。

你可能已经遇到过这些情况：
- AI写的代码跑不起来
- 游戏能打开，但功能不对
- 浏览器弹出一堆看不懂的红色错误
- AI说它做好了，但根本没实现你要的功能

**这些都很正常！**

今天这一章，就专门教你怎么应对这些情况。

---

## 三种常见的"出错"

先分清楚是什么问题：

### 类型1：AI没理解你的意思

**症状：** 代码能跑，但做的东西不是你想要的。

比如你说"做一个计算器"，AI给你做了一个只能算加法的。

**原因：** 你的Prompt没说清楚。

**解决：** 补充细节，再说一遍。

---

### 类型2：代码有bug

**症状：** 代码能跑起来，但行为不对。

比如贪吃蛇吃到食物不变长，或者分数不增加。

**原因：** AI写的代码逻辑有问题。

**解决：** 告诉AI哪里不对，让它修。

---

### 类型3：代码根本跑不起来

**症状：** 打开网页一片空白，或者浏览器控制台一堆红色报错。

**原因：** 代码有语法错误，或者复制的时候漏了一部分。

**解决：** 把错误信息发给AI，让它帮你找问题。

---

## 调试三板斧

遇到问题就用这三招：

### 第1招：把错误发给AI

如果浏览器有报错，把报错信息复制下来，连同你的代码一起发给AI。

怎么看报错？按 `F12` 打开开发者工具，点 Console（控制台），红色的字就是错误信息。

发给AI的格式：

```
我的代码运行时报错了，错误信息是：

[粘贴错误信息]

我的代码是：

[粘贴代码]

请帮我分析问题在哪，并修复。
```

### 第2招：描述"预期vs实际"

如果代码能跑但功能不对，告诉AI：
- 你期望发生什么
- 实际发生了什么

发给AI的格式：

```
我的贪吃蛇游戏有问题：

预期：吃到食物后蛇会变长
实际：吃到食物后蛇长度没变

我的代码如下：

[粘贴代码]

请帮我找出问题并修复。
```

### 第3招：简化问题

如果问题很复杂、定位不了，试着把问题简化。

比如：
- 把复杂代码里的一部分注释掉，看还出不出错
- 让AI重新写一个最简版本，再一点点加功能
- 一次只改一个地方，看哪一步出的问题

## 真实案例

来看几个真实的调试案例：

### 案例1：贪吃蛇不动

**问题描述：**
打开网页后，蛇一动不动。

**可能原因：**
1. 游戏循环没有启动
2. 键盘事件没有绑定

**发给AI：**
```
我的贪吃蛇游戏打开后蛇不动，按方向键也没反应。代码如下：

[粘贴代码]

请检查游戏循环和键盘事件是否正确。
```

### 案例2：网页完全空白

**问题描述：**
打开HTML文件，完全空白。

**检查步骤：**
1. 按F12打开控制台，看有没有红色报错
2. 检查代码是不是完整的（有没有漏复制）
3. 检查HTML标签是不是成对出现的（如 `<html>` 和 `</html>`）

**发给AI：**
```
我的网页打开后一片空白。控制台报错如下：

[粘贴错误]

代码如下：

[粘贴代码]

请帮我修复。
```

### 案例3：功能部分不工作

**问题描述：**
游戏能玩，但"重新开始"按钮点了没反应。

**发给AI：**
```
我的贪吃蛇游戏里，游戏结束后"重新开始"按钮点了没反应。

代码如下：

[粘贴代码]

请检查重新开始按钮的事件绑定是否正确。
```

## 调试心法

**心法1：别慌**

出错是正常的。专业程序员的日常就是和bug打交道。

**心法2：仔细读报错**

错误信息里通常会告诉你问题在哪一行。虽然可能看不懂具体原因，但行号是有用信息。

**心法3：一次改一个地方**

如果同时改很多东西，出了问题你都不知道是哪个改动导致的。

**心法4：没事多保存**

在代码能跑的时候保存一份。万一改坏了，还能回到上一个版本。

**心法5：实在不行，重来**

如果代码已经被改得一团糟，有时候最快的方法是让AI重新生成一遍。

## 调试挑战

来试试你的调试能力！

**挑战1：故意让代码出错**

把你的贪吃蛇代码随便删掉一行，保存，看看会报什么错。

试着：
1. 看懂错误信息
2. 用刚学的方法让AI帮你修

**挑战2：体验完整调试流程**

让AI帮你做一个"倒计时器"：
- 设定时间，点击开始，倒数到0
- 到0时弹出提示

如果第一次生成的代码不完美，用今天学的方法调试，直到它完全正常。

**挑战3：收集你的Bug案例**

以后每次遇到bug，记下来：
- 什么问题
- 怎么发现的
- 怎么解决的

积累多了你就成调试高手了！

---

## 划重点！

1. AI出错很正常，别慌
2. 分清三种情况：没理解意思、代码有bug、代码跑不起来
3. 调试三板斧：发错误给AI、描述预期vs实际、简化问题
4. 养成好习惯：一次改一个地方，多保存
5. 实在不行就重来，这也是一种策略

调试是编程的核心技能之一。学会了这个，你就能应对大多数问题！
