# 学会跟AI说话

---

## 点外卖 vs 跟AI说话

你点过外卖吗？想象一下：

**这样点外卖：**
```
老板，我饿了
```
老板会懵：你要吃啥？甜的咸的？多少钱的？送哪里？

**这样点外卖：**
```
老板，我要一份兰州拉面，面硬一点，不要香菜，打包，送到xx小区3号楼
```
老板秒懂，完美送达！

跟AI说话也是一样的道理。

你发给AI的话，有个名字叫 **Prompt（提示词）**。但名字不重要，重要的是：

**你说得越清楚，AI给的东西越靠谱。**

---

## 翻车现场：亲自体验

来，我们做个实验。

**实验1：模糊 vs 清晰**

先发这个：
```
帮我写代码
```

再发这个：
```
帮我用HTML写一个网页，网页上显示"Hello World"这几个字，字要大一点
```

对比一下两次的回答，哪个更有用？

**实验2：没背景 vs 有背景**

先发这个：
```
解释什么是变量
```

再发这个：
```
我是一个12岁的学生，从来没学过编程。请用最简单的话解释什么是变量，最好举个生活中的例子
```

哪个解释你更能听懂？

## 说清楚4件事

跟AI说话，记住这4件事：

### 1. 你要它扮演谁（角色）

告诉AI它是谁，它就会用那个身份回答你。

例子：
- "你是一个编程老师"
- "你是一个游戏设计师"
- "你是一个10岁的小朋友"（测试它解释得够不够简单）

### 2. 你要它做什么（任务）

说清楚你要什么东西。

例子：
- "帮我写一个贪吃蛇游戏"
- "解释什么是循环"
- "帮我检查这段代码有没有错"

### 3. 有什么要求（限制）

说清楚大小、颜色、风格、用什么技术等等。

例子：
- "用HTML和JavaScript"
- "游戏窗口400x400像素"
- "蛇是绿色的，食物是红色的"
- "所有代码写在一个文件里"

### 4. 有没有参考（例子）

如果能给AI一个参考，它更容易理解你要什么。

例子：
- "就像手机上的计算器那样"
- "类似微信的聊天界面"
- "参考这个格式：xxx"

## 实战：写一个好Prompt

来，我们用刚学的方法，写一个让AI帮我们做贪吃蛇游戏的Prompt：

```
你是一个游戏开发高手。

请帮我做一个贪吃蛇游戏。

要求：
- 用HTML + JavaScript，所有代码放一个文件里
- 游戏窗口400x400像素
- 蛇是绿色方块，食物是红色圆点
- 用键盘方向键控制蛇移动
- 吃到食物蛇会变长，分数+1
- 撞墙或撞到自己就游戏结束
- 游戏结束显示分数，可以重新开始

做出来的效果类似于谷歌搜索"贪吃蛇"时出现的小游戏。
```

看，是不是4件事都说清楚了？

## Prompt医生：找出问题

下面这些Prompt都有毛病，你能诊断出问题吗？

**病号1：**
```
写个网站
```
问题在哪？缺了什么？

**病号2：**
```
帮我写一个计算器，要好看
```
"好看"是什么意思？AI怎么知道你觉得什么好看？

**病号3：**
```
你是一个Python专家，帮我用JavaScript写一个游戏
```
角色和任务对不上，这会让AI困惑。

---

## 动手改一改

把这些"病号"Prompt改成健康的版本：

**改造任务1：**

原版：
```
做个游戏
```

改成什么样才好？试试自己改，然后发给AI看看效果。

**改造任务2：**

原版：
```
解释循环
```

加上角色、背景、要求，让AI给出更适合你的解释。

## 挑战任务

**挑战：最短Prompt大赛**

用最少的字，让AI画出一只猫。

但要求：
- 必须是ASCII字符画（用符号画的）
- 猫要有耳朵、眼睛、胡须

你能用多少个字做到？和同学比一比！

---

## 划重点！

1. **Prompt = 你发给AI的话**，说得越清楚，结果越好
2. **4件事**：角色、任务、限制、例子
3. 不用每次都写全，但写清楚的地方越多，AI越不会乱来
4. **遇到AI给的东西不对**，先检查自己的Prompt是不是没说清楚
5. 好Prompt是练出来的，多试多改！
