## 各章改进要点

### 第1章：AI是你的超级队友
- 开场用故事/场景引入
- 增加"AI翻车时刻"（让AI算数出错、问实时信息不知道）
- 语气改为"试试看！""超酷！"
- 练习改为探索性任务

### 第2章：学会跟AI说话
- 用"点外卖"类比替代RICE框架
- 增加"翻车现场"对比实验
- 把"RICE"改为"说清楚4件事"
- 增加"Prompt医生"诊断练习

### 第3章：做一个贪吃蛇游戏（合并）
- 合并原3+4章，从规则分析到运行一章搞定
- 简化保存文件步骤
- 增加"常见翻车"急救站
- 增加"改装挑战"（换颜色、加暂停按钮）

### 第4章：做一个属于你的小工具
- 开场用"你有什么烦恼"引入
- 减少直接给Prompt模板，引导学生自己写
- 增加自主选择空间

### 第5章：让全世界看到你的作品
- 部署步骤从11步简化为3步核心步骤
- 增加"不部署也没关系"的说明
- 常见问题前置

### 第6章：当AI出错了怎么办（新增）
- 错误类型分类（代码bug、AI没理解、运行报错）
- 调试三板斧（发错误给AI、一步步检查、简化问题）
- 真实案例分析
- 调试挑战练习

### 第7章：自由创作时间（合并）
- 合并原7+8章
- 减少模板，增加自由度
- 灵感激发 → 自主选择 → 创作 → 展示
- 继续学习资源

## 写作风格指南

### 语气
| 不要 | 要 |
|------|------|
| 请您... | 试试... |
| 同学们需要... | 你可以... |
| 本章将学习... | 这一章，我们要搞定... |

### 格式要求
- 每章必须有：开场钩子、动手环节、翻车预警、挑战任务、划重点
- 每段不超过3-4行
- 重要内容用**粗体**
- 代码块前要有引导语
