## 📖 课程简介

### 什么是 Vibe Coding？

**Vibe Coding** 是由前 Tesla AI 总监、OpenAI 联合创始人 **Andrej Karpathy** 于 2025 年提出的全新编程理念：

> "完全沉浸于氛围中，拥抱指数级增长，甚至忘记代码的存在"

在传统编程中，你需要记住大量语法、处理复杂的报错信息、花费大量时间调试。而在 **Vibe Coding** 时代，你只需要：

1. 🗣️ **用自然语言描述你想要什么**
2. 🤖 **让 AI 帮你生成代码**
3. ✨ **专注于创意和产品本身**

### 为什么选择这门课程？

| 传统编程学习 | Vibe Coding 学习 |
|-------------|-----------------|
| 需要记忆大量语法 | AI 帮你处理语法细节 |
| 报错信息看不懂 | AI 解释错误并指导修复 |
| 从 0 到作品需要几个月 | **第一节课就能做出作品** |
| 门槛高，容易放弃 | 成就感强，越学越有趣 |

## 📘 模块一：认识 AI 编程（2 课时）

### 学习目标

- 理解什么是人工智能和 Vibe Coding
- 完成 AI 工具的注册和配置
- 进行第一次 AI 对话

### 第 1 课：AI 与 Vibe Coding 初探

#### 知识要点

**什么是人工智能？**

人工智能 (AI) 就像是一个非常聪明的助手，它通过学习大量的数据，学会了理解人类的语言并完成各种任务。

```
人类 🧑 ──「请帮我写一首关于春天的诗」──▶ AI 🤖 ──▶ 生成诗歌 📝
```

**什么是 Vibe Coding？**

Vibe Coding 是一种新的编程方式：
- 传统编程：你需要一行一行写代码
- Vibe Coding：你用自然语言告诉 AI 你想要什么，AI 帮你写代码

#### 课堂活动

1. **AI 应用大搜索**：说出你知道的 AI 应用（抖音推荐、搜索引擎、智能客服...）
2. **观看视频**：了解 Andrej Karpathy 如何定义 Vibe Coding
3. **讨论**：AI 能帮程序员做什么？AI 不能做什么？

### 第 2 课：工具配置与第一次对话

#### 实践步骤

**步骤 1：注册 AI 工具**

选择以下任一工具完成注册：
- Trae：https://www.trae.ai （推荐国内学生）
- 智谱清言：https://chatglm.cn
- 通义千问：https://tongyi.aliyun.com

**步骤 2：第一次 AI 对话**

尝试向 AI 发送以下请求：

```
你好！请用一段话介绍一下你自己，以及你能帮我做什么？
```

**步骤 3：创意体验**

让 AI 帮你完成一个有趣的任务：

```
请帮我用 emoji 画一幅春天的风景画。
```

#### 课后挑战

- [ ] 尝试用 3 种不同的方式让 AI 自我介绍
- [ ] 让 AI 帮你写一个自我介绍

## 📘 模块二：提示词艺术（3 课时）

### 学习目标

- 理解什么是提示词 (Prompt)
- 掌握 RICE 提示词框架
- 学会引导 AI 给出更好的回答

### 核心概念：什么是 Prompt？

**Prompt（提示词）** = 你发给 AI 的消息

AI 的回答质量 = 你的 Prompt 质量

```
模糊的 Prompt ──▶ 模糊的回答 ❌
清晰的 Prompt ──▶ 精准的回答 ✅
```

### 第 3 课：好 Prompt vs 坏 Prompt

#### 对比实验

| ❌ 坏 Prompt | ✅ 好 Prompt |
|-------------|-------------|
| 帮我写代码 | 帮我用 Python 写一个九九乘法表程序 |
| 写一个游戏 | 用 JavaScript 写一个贪吃蛇游戏，要求画布 400x400 像素，用方向键控制 |
| 解释一下循环 | 请用通俗易懂的语言，向一个 12 岁的学生解释什么是 for 循环，并给出一个生活中的例子 |

#### 课堂练习

改写以下 Prompt，使其更清晰：

1. "帮我做个网站" → ？
2. "写个计算器" → ？
3. "解释变量" → ？

### 第 4 课：RICE 提示词框架

#### RICE 框架详解

```
┌─────────────────────────────────────────────────────────────┐
│  R - Role（角色）                                            │
│  告诉 AI 它应该扮演什么角色                                   │
│  例：你是一个专业的游戏开发工程师                              │
├─────────────────────────────────────────────────────────────┤
│  I - Instruction（指令）                                     │
│  清楚说明你想要 AI 做什么                                     │
│  例：请帮我创建一个贪吃蛇游戏                                  │
├─────────────────────────────────────────────────────────────┤
│  C - Context（背景/限制）                                    │
│  提供必要的背景信息和限制条件                                  │
│  例：使用 HTML+CSS+JS，画布 400x400，纯绿色蛇身               │
├─────────────────────────────────────────────────────────────┤
│  E - Example（示例）                                         │
│  如果可能，提供一个参考示例                                    │
│  例：类似于这个网站的效果：[链接]                              │
└─────────────────────────────────────────────────────────────┘
```

#### 完整示例

```text
【R - 角色】
你是一个专业的前端游戏开发工程师，擅长用 HTML5 Canvas 开发小游戏。

【I - 指令】
请帮我创建一个经典的贪吃蛇游戏。

【C - 背景/限制】
- 使用 HTML + CSS + JavaScript，所有代码写在一个 HTML 文件中
- 游戏画布大小：400x400 像素
- 蛇的颜色：绿色方块
- 食物的颜色：红色圆形
- 使用键盘方向键控制
- 需要显示当前分数
- 游戏结束后显示最终分数并可以重新开始

【E - 示例】
类似于 Google 搜索"贪吃蛇"时出现的小游戏效果。
```

### 第 5 课：让 AI 扮演角色

#### 角色扮演技巧

让 AI 扮演特定角色，可以获得更专业、更有针对性的回答：

| 角色设定 | 适用场景 |
|----------|----------|
| "你是一个耐心的编程老师" | 学习新概念时 |
| "你是一个代码审查专家" | 检查代码问题时 |
| "你是一个产品经理" | 规划功能需求时 |
| "你是一个 12 岁的学生" | 测试解释是否通俗易懂 |

#### 实践练习

使用角色扮演，让 AI 教你一个概念：

```
你是一个超级有耐心的编程老师。我是一个从来没学过编程的 12 岁学生。

请用最简单、最通俗的语言，向我解释什么是"变量"。

要求：
1. 使用生活中的例子
2. 举至少 2 个例子
3. 最后用一句话总结
```

## 📘 模块三：AI 做小游戏（4 课时）⭐

### 第 6 课：网页基础入门

#### 网页三剑客

```
┌─────────────────────────────────────────────────────────────┐
│  HTML  - 骨架                                               │
│  定义网页的结构和内容（标题、段落、按钮...）                   │
├─────────────────────────────────────────────────────────────┤
│  CSS   - 皮肤                                               │
│  定义网页的样式（颜色、大小、位置...）                        │
├─────────────────────────────────────────────────────────────┤
│  JavaScript - 灵魂                                          │
│  定义网页的行为（点击事件、动画、游戏逻辑...）                 │
└─────────────────────────────────────────────────────────────┘
```

#### 实践：用 AI 生成第一个网页

**Prompt：**

```
请帮我创建一个简单的 HTML 网页，包含：
1. 一个大标题写着"我的第一个网页"
2. 一段自我介绍文字
3. 一个按钮，点击后弹出"你好！"

请把所有代码写在一个 HTML 文件中。
```

### 第 7 课：贪吃蛇游戏设计

#### 游戏设计思路

在写代码之前，先想清楚游戏的规则：

**贪吃蛇规则：**
1. 蛇会自动向前移动
2. 玩家用方向键控制蛇的方向
3. 蛇吃到食物后身体变长，分数增加
4. 蛇撞到墙壁或自己的身体，游戏结束

#### 绘制草图

在纸上画出游戏界面：

```
┌──────────────────────────────┐
│  分数: 0                      │
├──────────────────────────────┤
│                              │
│      🟩🟩🟩🟩                 │
│              🔴               │
│                              │
│                              │
└──────────────────────────────┘
```

### 第 8 课：AI 生成贪吃蛇

#### 完整 Prompt

```text
你是一个专业的游戏开发工程师。请帮我用 HTML + CSS + JavaScript 创建一个经典的贪吃蛇游戏。

要求：
1. 所有代码写在一个 HTML 文件中，方便我直接打开运行
2. 游戏画布大小：400x400 像素，背景为深色
3. 蛇用绿色方块表示，每个方块 20x20 像素
4. 食物用红色圆形表示
5. 使用键盘方向键（↑↓←→）控制蛇的移动方向
6. 蛇吃到食物后身体变长，同时生成新的食物
7. 蛇撞到墙壁或撞到自己的身体，游戏结束
8. 页面顶部显示当前分数
9. 游戏结束后显示"Game Over"和最终分数，并有"重新开始"按钮
10. 游戏速度适中，让玩家有反应时间

请确保代码可以直接运行，不需要任何额外的库或框架。
```

#### 运行测试

1. 将 AI 生成的代码复制到文本编辑器
2. 保存为 `snake.html`
3. 用浏览器打开文件
4. 开始游戏！

### 第 9 课：美化与完善

#### 迭代改进

游戏能运行后，我们来让它更好玩：

**Prompt 1：添加音效**
```
请为我的贪吃蛇游戏添加音效：
1. 吃到食物时发出"叮"的声音
2. 游戏结束时发出"咚"的声音
请使用 Web Audio API 生成简单的音效，不需要外部音频文件。
```

**Prompt 2：美化界面**
```
请美化我的贪吃蛇游戏界面：
1. 添加渐变背景
2. 蛇身使用渐变绿色，从头到尾由深变浅
3. 添加游戏标题
4. 美化分数显示和按钮样式
```

**Prompt 3：增加难度**
```
请修改贪吃蛇游戏，让难度随分数增加：
- 每吃 5 个食物，游戏速度加快 10%
- 在界面上显示当前速度等级
```

#### 分享你的作品

- 把 `snake.html` 发给朋友
- 或者上传到 GitHub，生成在线链接

### 第 11-12 课：实现你的工具

#### 番茄钟示例 Prompt

```text
你是一个前端开发专家。请帮我创建一个番茄钟（Pomodoro Timer）应用。

功能要求：
1. 默认工作时间 25 分钟，休息时间 5 分钟
2. 一个大的倒计时显示，格式为 MM:SS
3. 三个按钮：开始、暂停、重置
4. 工作时间结束后自动切换到休息时间
5. 切换时发出提示音
6. 显示当前是"工作中"还是"休息中"
7. 记录今天完成的番茄数量

界面要求：
1. 简洁现代的设计风格
2. 工作时背景为柔和的红色
3. 休息时背景为柔和的绿色
4. 大字体显示时间，易于阅读

请把所有代码写在一个 HTML 文件中。
```

#### 单词卡片示例 Prompt

```text
请帮我创建一个单词记忆卡片应用。

功能要求：
1. 预设 20 个英语单词和中文翻译

3. 点击卡片可以翻转，看到另一面

5. 有"下一个"按钮切换单词

1. 卡片有翻转动画效果
2. 卡片大小适中，居中显示
3. 简洁美观的设计
```

### 第 14-15 课：AI 生成个人网站

#### 完整网站 Prompt

```text
你是一个专业的前端开发工程师。请帮我创建一个个人展示网站。

个人信息：
- 姓名：[你的名字]
- 身份：[年级]学生
- 兴趣：编程、[其他兴趣]
- 邮箱：[你的邮箱]

网站结构：
1. 导航栏：Logo + 首页/关于/作品/联系
2. Hero 区域：大背景图 + 欢迎语 + 简短介绍
3. 关于我：一段自我介绍
4. 我的技能：展示会的技术（可以用进度条）
5. 我的作品：展示 3 个项目（贪吃蛇等）
6. 联系方式：展示邮箱和社交链接
7. 页脚：版权信息

设计要求：
1. 现代简约风格
2. 使用渐变色和阴影效果
3. 响应式设计，手机也能看
4. 平滑滚动效果
5. 导航栏固定在顶部

请把所有代码写在一个 HTML 文件中，包含 CSS 和 JavaScript。
```

### 第 16 课：部署上线

#### 方案一：GitHub Pages（推荐）

1. 注册 GitHub 账号
2. 创建新仓库，命名为 `username.github.io`
3. 上传你的 `index.html` 文件
4. 访问 `https://username.github.io` 查看网站

#### 方案二：Vercel（更简单）

1. 访问 vercel.com
2. 用 GitHub 账号登录
3. 导入你的代码仓库
4. 自动部署完成！

### 第 18-19 课：项目开发

#### 开发流程

```
需求分析 ──▶ 功能拆解 ──▶ AI 生成代码 ──▶ 测试运行 ──▶ 迭代改进
    │           │              │             │             │
    ▼           ▼              ▼             ▼             ▼
 想清楚要    分成小任务    一步步让 AI   发现问题    修复 Bug
 做什么      逐个完成      帮你写代码    记录下来    添加功能
```

#### 遇到问题怎么办？

1. **代码报错** → 把错误信息发给 AI，让它帮你分析
2. **功能不对** → 向 AI 描述预期效果和实际效果的差异
3. **不知道怎么实现** → 让 AI 先给你解释思路，再写代码

### 第 20 课：成果展示

#### 展示环节

每位同学有 5-10 分钟展示自己的作品：

1. **介绍**：项目名称和功能
2. **演示**：实际操作展示
3. **分享**：遇到的困难和解决方法
4. **反思**：学到了什么

#### 互评环节

给其他同学的作品打分（1-5 分）：
- 创意性：想法是否有趣新颖？
- 完成度：功能是否完整？
- 美观度：界面是否好看？
- 实用性：是否真的有用？
